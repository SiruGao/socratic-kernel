# 模型中心

模型中心不把“模型接入”表现为一个复杂表单，而是拆成三个明确层级：

```text
浏览器应用
    ↓
Socratic Kernel Gateway
    ↓
模型供应商与具体模型
```

## 四个概念

### 1. 本地追问

不调用任何大模型。问题由浏览器中的规则引擎生成，可以离线使用，也不会发送当前审议内容。

### 2. Gateway

Gateway 是浏览器与模型供应商之间的服务端中转层。它负责：

- 保存供应商 API Key；
- 验证浏览器请求；
- 列出已经配置的供应商；
- 调用用户选择的模型；
- 规范化模型返回格式；
- 在错误时返回可处理的状态。

供应商 API Key 不能放在浏览器、PWA、桌面安装包或移动安装包中。

### 3. 供应商

供应商是 OpenAI、Anthropic、Google、DeepSeek、阿里云百炼、Moonshot 或 xAI 等模型服务平台。

模型中心只展示 Gateway 已经成功配置的供应商。用户不会在网页中填写供应商 API Key。

### 4. 默认模型

每个供应商可以在服务端配置一个默认模型。模型中心显示供应商名称和实际模型 ID，并允许用户选择新审议默认使用哪个模型。

## 三步设置

### 第一步：连接 Gateway

点击“检查 Gateway”只验证：

- 地址能否访问；
- 是否部署了 Socratic Kernel Gateway；
- Gateway 返回了多少个已配置供应商；
- 是否要求额外访问令牌。

这个动作不会调用模型，通常也不会产生模型费用。

### 第二步：选择默认模型

供应商目录显示：

- 供应商名称；
- 实际模型 ID；
- 服务端是否已配置；
- 当前默认选择。

没有出现在目录中的供应商，说明 Gateway 环境变量中没有相应密钥，或者模型配置无效。

### 第三步：启用并测试

“启用 AI 追问”只改变新审议的默认引擎。用户仍可以在每次审议前选择本地追问。

“测试所选模型”会发起一次真实的最小请求，用于验证：

- Gateway 访问令牌；
- 供应商 API Key；
- 模型名称；
- 账户余额和权限；
- 模型是否能返回结构化问题。

真实测试可能产生少量模型费用。

## 高级连接设置

高级设置默认折叠，因为多数用户不需要修改。

### Gateway 地址

- 留空：使用当前网站同域的 `/api`；
- 填写完整 HTTPS 地址：当前网页与 Gateway 分开部署，例如 GitHub Pages 前端连接 Vercel Gateway。

### Gateway 访问令牌

这是 Socratic Kernel Gateway 的访问密码，不是供应商 API Key。它只用于限制谁可以使用部署者承担费用的 Gateway。

## 状态含义

| 状态 | 含义 | 下一步 |
| --- | --- | --- |
| 本地追问 | 未启用 AI 或 Gateway 不可用 | 可直接使用；需要 AI 时连接 Gateway |
| Gateway 未连接 | 地址错误、没有部署或网络不可达 | 检查 Vercel 部署和地址 |
| Gateway 已连接但 0 个供应商 | 服务端可达，但没有供应商密钥 | 在 Vercel 环境变量中配置至少一个供应商 |
| 供应商已配置 | Gateway 能识别该供应商 | 选择模型并进行真实测试 |
| 模型测试失败 | 密钥、余额、模型名、令牌或格式有问题 | 根据页面错误提示检查对应配置 |
| 模型测试通过 | 完整调用链路正常 | 保存并启用 AI 追问 |

## 数据边界

AI 追问只发送当前审议的：

- 模式；
- 问题；
- 初始立场；
- 当前依据；
- 确信度；
- 挑战强度。

默认不发送：

- 历史审议档案；
- 其他问题的回答；
- 长期模式统计；
- 导出文件；
- 供应商 API Key。

## 设计来源

信息架构借鉴了成熟网关控制台的做法：把 Gateway 连接、认证、供应商目录、默认模型、状态检查和高级配置分离，而不是把所有字段平铺在同一个表单中。

Socratic Kernel 没有照搬其他产品的视觉组件。模型中心继续使用本项目的编辑式排版、骨白纸张、墨黑、深酒红和铜金视觉系统。
