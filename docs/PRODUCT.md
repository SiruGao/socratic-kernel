# Socratic Kernel 产品原则

## 产品使命

Socratic Kernel（内核）不是答案生成器，而是一套认知自主权练习系统。它帮助用户在使用 AI、阅读信息、形成观点和作出决定时，保留问题定义、价值排序、最终判断和责任承担。

## 核心闭环

1. 用户先写下自己的问题、立场、依据和确信程度。
2. 系统识别可能存在的认知线索，但不把线索当作人格诊断。
3. 系统按概念、证据、可证伪性、最强反方、价值、后果和责任进行追问。
4. 用户逐题回答，不能直接领取最终结论。
5. 用户用自己的语言形成暂时判断、下一步行动和愿意承担的代价。
6. 系统记录纵向模式，但用户始终拥有查看、导出、修正和删除权。

## 不可妥协的原则

- **人的初步判断在前。** 系统不能在用户尚未形成任何立场时直接占据问题空间。
- **纠正推理，不裁决人生。** 系统可以指出事实错误、逻辑矛盾和证据缺口，但不能把某一价值框架包装成客观真理。
- **反迎合不等于唱反调。** 每次挑战必须说明对象和理由，不为了表现聪明而制造摩擦。
- **最终判断归还用户。** 重要审议必须以用户自己的结论和责任声明结束。
- **记忆必须可见、可控。** 不建立隐蔽人格画像，不将精神数据用于广告、说服优化或第三方交易。
- **成功不是依赖。** 核心指标是用户是否更能独立提出反例、识别外包边界并承担判断。

## 当前版本边界

当前版本是无服务端、本地优先的产品验证版。它使用可检查的规则与结构化追问，不声称拥有心理诊断能力，也不替代心理治疗、法律、医疗或财务专业意见。
