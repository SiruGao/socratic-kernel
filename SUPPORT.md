# Support

感谢你使用 Socratic Kernel。为了让问题更快得到处理，请选择合适的入口。

## 使用与部署问题

优先阅读：

- [`docs/QUICKSTART.md`](./docs/QUICKSTART.md)
- [`docs/MODEL_CENTER.md`](./docs/MODEL_CENTER.md)
- [`docs/AI_GATEWAY.md`](./docs/AI_GATEWAY.md)
- [`docs/DEPLOYMENT.md`](./docs/DEPLOYMENT.md)

仍无法解决时，提交 Bug Report，并提供：

- 部署方式；
- 浏览器和设备；
- 最小复现步骤；
- 已移除密钥和隐私内容的截图或日志。

## 功能建议

使用 Feature Request。建议需要说明：

- 真实使用场景；
- 对用户主体性的影响；
- 数据与隐私影响；
- 是否存在更简单的方案；
- 如何判断功能真正有效。

## 哲学、方法与产品方向

开放性讨论更适合 GitHub Discussions，而不是 Bug Issue，例如：

- AI 什么时候应该反驳用户；
- 什么判断不能外包；
- 一种哲学方法如何转换为可检查的产品协议；
- 如何评估用户是否变得更独立。

仓库管理员开启 Discussions 后，README 和 Issue 模板会补充直接入口。

## 安全与隐私漏洞

不要公开提交：

- API Key 或 Gateway 访问令牌；
- 可以被利用的安全细节；
- 真实用户的私密审议正文；
- 未经许可的个人身份信息。

请使用 GitHub Private Vulnerability Reporting：

```text
https://github.com/SiruGao/socratic-kernel/security/advisories/new
```

并阅读 [`SECURITY.md`](./SECURITY.md)。

## 不提供的支持

Socratic Kernel 不是心理治疗、医疗、法律或财务建议服务。项目维护者无法替用户判断个人生活问题，也不会通过 Issue 分析用户人格或心理状态。
