# Code of Conduct

Socratic Kernel 的目标是帮助人更诚实地思考。项目社区也应当以同样的标准对待彼此：可以严格讨论观点，但不能贬低表达观点的人。

## 我们鼓励

- 对事实、论证和设计提出具体批评；
- 主动说明自己的假设和不确定性；
- 先理解对方最强的观点，再进行反驳；
- 对不同文化、语言、身份和经验保持尊重；
- 承认错误并根据新证据修正立场；
- 给新贡献者提供可执行的反馈。

## 不可接受

- 人身攻击、羞辱、威胁或持续骚扰；
- 基于身份特征的歧视性表达；
- 公开他人的私人信息；
- 性骚扰或不受欢迎的性化内容；
- 故意曲解他人观点并反复挑衅；
- 利用社区讨论测试或改进操纵、欺骗和心理控制技术；
- 在被要求停止后继续进行不受欢迎的私人联系。

## 讨论原则

严格不等于敌意。提出批评时，请尽量包含：

1. 你认为存在的问题；
2. 支持判断的证据或复现方式；
3. 可能的替代解释；
4. 一个可执行的改进建议。

## 执行

发现违反行为准则的内容时，可以通过维护者的 GitHub 个人资料联系，或在不包含敏感细节的情况下提交管理 Issue。

维护者可以根据严重程度采取提醒、删除内容、限制参与或永久禁止等措施。涉及安全漏洞或私人信息时，请按照 [`SECURITY.md`](./SECURITY.md) 使用私密报告渠道。

本行为准则适用于项目仓库、Issue、Pull Request、讨论区以及代表项目参与的其他公开空间。
